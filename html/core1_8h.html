Detailed Description

Task declarations for Core 1 of the Smart Security Door System on ESP32.

This header provides declarations for FreeRTOS tasks that execute on Core 1. These tasks include sensor polling, RFID scanning, and access control logic. It complements core1.cpp and is intended to be used alongside shared definitions in global_defs.h.

Task Overview:

  • sensorReadTask reads PIR and ultrasonic sensor values periodically.
  • sensorProcessTask aggregates data from sensors and determines if backlight or unlock actions should be triggered.
  • taskRFIDReader continuously checks for new RFID cards and sends UIDs.
  • taskPrinter validates RFID UIDs, manages lock state, and provides user feedback.
  • distanceTask and rtcTask are deprecated debug routines.

◆ sensorProcessTask()

void sensorProcessTask ( void * pvParameters)

Processes buffered sensor readings and triggers appropriate system behavior.

  • Computes rolling average of distance and motion buffer.
  • Determines close_dist and motion_detected flags.
  • Turns on LCD backlight on detection and logs event using RTC with semaphore.

◆ sensorReadTask()

void sensorReadTask ( void * pvParameters)

Periodically triggers ultrasonic distance measurements and reads PIR sensor state.

  • Triggered every 20ms (50Hz).
  • Sends sensor readings (sensorData_t) to sensorQueue.
  • Uses task notifications for echo pulse timing instead of pulseIn().

◆ taskPrinter()

void taskPrinter ( void * pvParameters)

Processes UIDs from rfidQueue and manages access control.

  • Compares received UID to allowed list.
  • If authorized:
    • Unlocks system.
    • Notifies ServoRunTask.
    • Starts/reset lock and backlight timers.
  • If unauthorized:
    • Logs timestamp and denies access.
  • Avoids redundant unlocks from repeated scans.

◆ taskRFIDReader()

void taskRFIDReader ( void * pvParameters)

Polls the RFID reader for new tag UIDs and sends them to rfidQueue.

  • Checks for new cards using MFRC522 interface.
  • Extracts and formats UID into a string and sends to queue.
  • Ensures card halting and crypto session termination.
